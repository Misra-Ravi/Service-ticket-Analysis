Weekly Incident Report Processor

Transforms a raw ServiceNow SAPUI5 export into a business-ready Excel workbook — aging, priority tiers, and carry-forward update logs, automated.

Python · openpyxl Excel VBA ServiceNow SAPUI5

Problem → Solution

Raw export gaps
  • No aging column — cannot see how old a ticket is
  • No business priority — SAP Priority ≠ review priority
  • No running log — meeting notes lost week to week
  • All 2,000+ rows visible — active tickets buried
What this adds
  • Aging and Last Update calculated in days (UTC-correct)
  • Customer Priority derived from Aging + Status + SAP Priority
  • Customer Update and SAP Update columns persist each week
  • Active tickets sorted to top, closed rows below

Columns with darker header background are new additions (locked except Customer Update and SAP Update). All original ServiceNow data is preserved. Active tickets sorted to top; 1,954 closed rows follow below.

New Columns Added

ColumnTypeLogic
Aging new Integer (days) today − CREATED ON (UTC), parsed as UTC datetime
Last Update new Integer (days) today − UPDATED ON (UTC), parsed as UTC datetime
Customer Priority new VERY HIGH / HIGH / MEDIUM HIGH / blank All 3 conditions required — see table below
Customer Update new Free text (editable) Running log — carried forward from previous week via CASE key
SAP Update new Free text (editable) Running log — carried forward from previous week via CASE key

Customer Priority Logic

All three conditions must be true simultaneously. Missing any one → blank.

PriorityAging thresholdStatusSAP Priority
VERY HIGH > 7 days In-scope Very High → Red
HIGH > 30 days In-scope High → Yellow
MEDIUM HIGH > 60 days In-scope Medium → Green
blank Threshold not met — Low or condition missing

In-scope statuses: Sent to SAP Customer Action SAP Proposed Solution

Usage

Python CLI
Excel VBA
# Install pip install -r requirements.txt # Basic — no carry-forward python scripts/process_tickets.py --raw "caseList 27.xlsx" # With carry-forward from previous week python scripts/process_tickets.py \ --raw "caseList 27.xlsx" \ --prev "caseList 26 Updated.xlsx" # Override run date (testing / backdating) python scripts/process_tickets.py \ --raw "caseList 27.xlsx" \ --date 2026-07-09
' One-time: VBA Editor (Option+F11) → File → Import → vba/modJCITracker.bas ' Paste into ThisWorkbook module: Private Sub Workbook_Open() On Error Resume Next ThisWorkbook.Sheets("Update Log").Visible = xlSheetVeryHidden On Error GoTo 0 ThisWorkbook.Protect Password:="JCI2026", Structure:=True End Sub ' Weekly: update Config!B2 with new file path → click "Refresh This Week"

Carry-Forward — How Notes Persist

Week N → type notes in Customer Update / SAP Update
         ↓ save (Cmd+S or click Save Updates)
Update Log stores { CASE → notes } permanently (hidden sheet)

Week N+1 → drop new raw export, click Refresh
          ↓ match on CASE number
          ↓ notes reappear automatically on matched rows
          ↓ new cases get blank cells — ready for entry

Nothing is ever lost. Closed tickets' notes remain in the log indefinitely.
