# Sample Data

These are **fully synthetic** files for testing and exploration. No real customer data,
case numbers, system IDs, or personal information from any SAP customer is included.

| File | Description |
|------|-------------|
| `sample_raw_export.xlsx` | Simulated ServiceNow SAPUI5 raw export — 130 rows, 19 columns, realistic status and priority distribution |
| `sample_previous_week_updated.xlsx` | Simulated previous week's updated output — same 130 cases with Aging, Customer Priority, and sample Customer Update / SAP Update notes pre-filled for the 22 in-scope tickets |

## How to use

```bash
# Basic run — raw file only
python scripts/process_tickets.py \
    --raw sample_data/sample_raw_export.xlsx

# Full run — with carry-forward from previous week
python scripts/process_tickets.py \
    --raw  sample_data/sample_raw_export.xlsx \
    --prev sample_data/sample_previous_week_updated.xlsx \
    --date 2026-07-09
```

Output is written to the same folder as the raw file as `sample_raw_export Updated.xlsx`.

## What to expect

- **22 in-scope rows** visible at the top (Sent to SAP, Customer Action, SAP Proposed Solution)
- **108 closed rows** below (Confirmed, Confirmed Automatically, Not Sent to SAP, In Processing)
- **Aging and Last Update** calculated from the `--date` value (defaults to today)
- **Customer Priority** assigned where all 3 conditions are met
- **Customer Update / SAP Update** notes carried forward from the previous week file

## Protection password

The previous week updated file is protected with password `SAMPLE2026`
(same as the real workbook uses `JCI2026`). The sample password is intentionally
different to avoid confusion.

## Data generation

Files were generated by `scripts/generate_sample_data.py` using:
- Fake customer: `Acme Manufacturing Corp.` / ID `0000099999`
- Fake reporter IDs and names (not real SAP user IDs)
- Realistic SAP module names, component paths, and installation formats
- Date range matching the real export (2021–2026)
